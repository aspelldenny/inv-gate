## Edge cases / limitations found

- **Python `rglob()` traversal order** is OS-dependent. Pins captured on macOS arm64 (Python 3.14.3). On Linux/amd64 the file order may differ. MANIFEST §4 note 3 records this: parity tests for P002-P005 should compare order-insensitively or run on same platform as oracle pin.
- **INV-009 fires twice for `ghp_` line**: once as `github-pat` (prefix match) and once as `generic-entropy` (fallback matches `api_key = "ghp_..."` form). This is oracle behavior — not a bug. Rust port must replicate both firings or parity will diverge.
- **INV-010 runtime scan count**: clean run reports `3 files scanned` — `.git/config`, `.mcp.json` (if exists), `.claude/settings.local.json` (if exists). Count is env-dependent. Pin records "3" but this is the harness's temp dir; Rust parity test should not assert on this count.
- **`check-schema-safety.sh` closing brace removal**: when AuditLog model is deleted, git diff shows `-}` line. This line does NOT match `^-\s+(model|enum)\s+\w+|^-\s+\w+\s+\S+` (brace is not `word space word` format) — confirmed no false positive (anchor #12).
