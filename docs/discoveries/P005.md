## Anchor #18 — diff scripts/check-*.py vs golden

**check-hardcoded-secrets.py: DIFFERS** (significant changes):
- Golden: hardcodes `SRC_DIRS_JS = ["src"]`, `SRC_DIRS_PY = ["astro-service"]`
- Adapted: `SRC_DIRS_JS = os.environ.get(...)` (env-var overrideable), `SRC_DIRS_PY = []` by default (different scan surface)
- Golden: 7 patterns + 1 generic. Adapted: 5 patterns + Stripe + Slack + 1 generic (more patterns)
- Golden: ALLOWLIST_SUBSTRINGS without `c993dc1e`/`google/gemini`/`anthropic/claude`. Adapted: includes them
- Golden: reports `INV-009: PASS (0 hardcoded secrets)`. Adapted: `INV-009: PASS (0 hardcoded secrets, N files scanned)` (counts files)
- **Coverage impact on swap:** Adapted script scans fewer Python dirs (empty `SRC_DIRS_PY` vs `["astro-service"]` in golden) but has more prefix patterns (Stripe/Slack). For this repo (Rust, no `astro-service/`), impact is zero — no Python source dirs to scan.

**check-runtime-secrets.py: DIFFERS** (significant changes):
- Golden: `INFRA_TOP_LEVEL` has `Path("Dockerfile")` + `Path("docker-compose.production.yml")`. Adapted: has `Path("Dockerfile.nextjs")` + `Path("astro-service/Dockerfile")` + `Path("astro-service/docker-compose.yml")` (project-specific paths)
- Golden: more Stripe/Slack patterns in prefix list. Adapted: same + extended ALLOWLIST (CHANGEME, EXAMPLE, `${`, `<`)
- db-conn regex: both have min-length check; adapted has same 8+ chars minimum
- **Coverage impact on swap:** Adapted scans different infra paths — but the CRITICAL runtime files (`.git/config`, `.mcp.json`, `.claude/settings.local.json`) are identical in both. Infra file differences are project-specific (Dockerfile.nextjs vs Dockerfile). Exit code contract identical (0/1 only).

**Verdict:** Adapted scripts have diverged from golden in both pattern coverage and scan surface. The swap (python3 → binary) uses the RUST PORT of the GOLDEN patterns (P002/P003 parity byte-exact) — NOT the adapted python scripts. This means the binary check is actually closer to the golden oracle than the adapted scripts. No coverage loss for the core runtime-state check (`.git/config` etc.). Coverage change is: Stripe/Slack patterns from adapted are NOT in the Rust port (only golden patterns), which is intentional per parity-first method.
