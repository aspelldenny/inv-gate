## Assumptions WRONG (Tầng 2 self-adapted)

## Edge Cases / Limitations Found

- **Multi-extension ordering (order risk P003+):** Python collects files by iterating extensions in array order (`.ts` first, then `.tsx`, `.js`, `.jsx`). For the current fixture (only `.ts` files), the Rust WalkDir per-extension approach is equivalent. When P003+ introduces multi-extension directories, the iteration-per-extension vs filter-all-at-once difference may produce different file ordering. Documented here as the V2 Debate Log hook for P003+.
- **WalkDir sort vs Python rglob:** Python `Path.rglob()` on macOS is alphabetical depth-first. `walkdir::sort_by_file_name()` matches this. MANIFEST.md §4.3 note: "rglob order OS-dependent" — this is consistent on a single platform (macOS arm64 Darwin 25.5.0).
