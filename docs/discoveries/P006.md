## Assumptions in phiếu — WRONG / Surprises

1. **tokio `time` feature required** (not documented in rmcp README): rmcp 1.7.0 internally uses `tokio::time` for timeout mechanisms. Without `"time"` in tokio features, runtime panics: `"A Tokio 1.x context was found, but timers are disabled"`. Fixed: added `"time"` to tokio features in Cargo.toml. No behavior change to CLI paths.

2. **`tokio::runtime::Runtime::new()` does not exist** — Runtime has no `new()` associated function in tokio 1.x. Must use `Builder::new_current_thread().build()` or `Builder::new_multi_thread()`. Fixed immediately in serve.rs.

3. **`rmcp::Error` deprecated** — should use `rmcp::ErrorData`. Compiler warning → fixed.

4. **`enable_io()` on runtime Builder requires `net` tokio feature** — not in our pinned deps. Not needed (tokio::io::stdin/stdout are always available). Fixed: use `.enable_time()` only.

5. **Phiếu assumed `Runtime::new().block_on`** — wrong API. Actual pattern: `Builder::new_current_thread().enable_time().build()?.block_on(async { ... })`.
