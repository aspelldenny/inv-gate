## Probe (c) — Unreachability of exit-1 so empty-tree (anchor #8)

`p010_probe_c_1commit_empirical_no_finding`: 1-commit repo, valid schema.prisma → `check schema` → exit 0, no `❌` in output. Confirms: diff vs empty-tree contains only `+` (addition) lines; `DESTRUCTIVE_RE` requires leading `-` → no match → Branch D safe → exit 0.

**Conclusion:** BACKLOG brief specced a "1-commit repo + destructive → exit 1" test. This test cannot be written correctly: after the fix, a 1-commit repo can only have additive diff vs empty-tree. The spec was based on golden's broken fallback (which routed to Branch C anyway, so exit 1 was also unreachable in the old code). Replaced with: probe (a) routing C→D + probe (c) empirical no-finding. Chủ nhà saw this at approval gate via phiếu §Context reasoning.
